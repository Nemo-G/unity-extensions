# Unity Game Planner

## Overview

此技能提供完整的 Unity 游戏架构设计工作流程。它能够将用户的游戏设计需求转化为详细的架构文档，包括游戏设计文档和多个PRD文档。严格遵循知识库中的模板文档规范，确保架构设计的一致性和完整性。

## When to Use This Skill

使用此技能当：
- 需要为新的 Unity 游戏创建完整的游戏设计文档
- 需要制定详细的prd开发文档（根据模板动态生成）
- 需要分析游戏的核心循环、数据流、模块划分
- 需要进行资源需求分析和技术方案规划
- 需要确保开发流程的安全协作和质量控制

## Core Workflow

### Step 1: 需求确认与分析

在开始设计前，必须收集并确认游戏的关键信息：

**必要信息**：
- 游戏名称（英文，用于文件命名）
- 游戏类型（如 2D 平台跳跃、FPS、横版射击等）
- 核心玩法机制
- 目标平台
- 特殊技术要求

### Step 2: 读取模板文档（必选）

**模板路径**：
游戏设计文档模板：`assets/template/design/`
prd文档模板： `assets/template/prd/`

### Step 3: 创建任务列表

使用 `todo_write` 创建文档编写任务列表：

**任务顺序**：
1. 游戏设计文档
2. 根据模板目录中的PRD模板文件顺序，依次创建对应的PRD文档
   - 动态读取规范文件名
   - 动态读取模板文件名
   - 按文件名排序或按模板中的序号排序
   - 为每个模板创建对应的文档编写任务

### Step 4: 创建游戏设计文档

**文件路径**：`context/{GameName}/游戏设计文档.md`

**文档内容**：
- 参考模板编写

### Step 5: 创建PRD文档

按照动态读取的模板顺序依次创建PRD文档：

**文件路径**：`context/{GameName}/{PRD的title}.json`

**命名规则**：使用PRD内容中的 `title` 字段作为文件名

**动态生成过程**：
1. 读取模板内容
2. 根据PRD的 `title` 字段生成文件名
3. 按模板顺序依次创建所有PRD文档

**关键约束**：
- ✅ 所有内容使用中文
- ❌ 严禁包含 C#、UXML、USS 或任何代码示例
- ❌ 严禁包含伪代码
- ❌ 严禁 UGUI 相关术语（使用 UI Toolkit）
- ✅ 保持与模板结构一致
- prd文档数量动态匹配模板，数量严格一致
- json结构严格保持和模板一致，只有任务内容可以修改
- **dependencies 字段必须严格遵守模板内容，不得自行修改或添加**：
- 严格遵守规范文档内的重要要求
- **场景搭建PRD必须包含 scene_config 字段**：
  - `path`: 场景文件路径，格式为 `Assets/Games/{GameName}/Scenes/{GameName}.unity`
  - `name`: 场景名称，通常为游戏名称

## Success Criteria

- ✅ 游戏设计文档已创建
- ✅ 所有PRD文档已创建（数量与模板匹配）
- ✅ 所有文档使用中文编写
- ✅ 无代码示例或 UGUI 术语
- ✅ 文档间依赖关系清晰
